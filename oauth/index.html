Login from an iframe

Scenario: You are on a given (e.g. posh.app) for a user who completed authentication previously.

  1. We start on the current domain and somehow prove our identity with posh.app
, which will set a server-side cookie and redirect to a third domain (here a store). This is a transparent operation to the user and note how quick it is.


Click here for "check and redirect".

Explanation: "Check and redirect" will first navigate to second domain that we control, which will check for the existence of a server-side cookie and redirect to a third domain (here a different store) with the cookie value (essentially transferring "state" from the second domain to the third).
